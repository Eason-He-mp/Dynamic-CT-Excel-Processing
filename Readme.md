# 动态 CT 扫描数据整理工具 (Dynamic CT Data Processor) 使用手册

## 1. 软件简介
本工具专为**原位动态 CT 扫描**实验设计。
在动态 CT 扫描过程中，上位机（记录时间、力、位移等）与 CT 控制机（生成 TIF 图像）通常独立运行，且可能存在时间差。本工具旨在通过精准的时间戳匹配，自动提取每次 CT 扫描（保载阶段）对应的力学数据，计算该时间段内的平均值，并生成最终的数据汇总报告。

### 🌟 核心功能
* **双模式支持**：兼容“连续共享边界帧（Type 1）”与“独立子文件夹（Type 2）”两种扫描数据结构。
* **全自动参数解析**：只需选择 TIF 文件夹，软件即可自动推算出扫描次数和单次扫描图片数。
* **智能过滤**：自动识别并排除 `di`（暗场）和 `io`（亮场）等探测器校准图像，无需人工剔除。
* **精准时间对齐**：支持自定义设备时间差，精准截取保载时间段内的力学数据并求平均值。
* **通用变量匹配**：不仅支持“力”和“位移”，也可自定义为“扭矩”、“应变”等任何所需变量。

## 2. 运行环境与安装
* **运行环境**：Windows10及以上操作系统（兼容 32 位与 64 位工业机）。
* **使用方法**：本软件为绿色免安装版。获取到 `CT_Data_Processor.exe` 文件后，**直接双击运行即可**，无需安装 Python 或任何额外环境。

---

## 3. 数据准备要求

在运行软件前，请确保您的数据符合以下规范：

### 模式一：Type 1 (主文件夹共享边界模式)
适用于连续扫描，相邻两次 Scan 共享同一张边界 TIF 图片。
* 所有实际的 TIF 图像需放在同一个**主文件夹**下。
* 主文件夹内必须包含一个名为 `recon` 的子文件夹，`recon` 内部需包含与扫描次数对应数量的子文件夹（用于软件自动计算扫描次数）。

### 模式二：Type 2 (子文件夹独立 Scan 模式)
适用于每次 Scan 独立保存的场景。
* 主文件夹下包含多个以 `4位数字` 开头的子文件夹（例如：`0001_scan`, `0002_data` 等）。
* 每个子文件夹代表一次独立的 Scan，内部存放该次扫描的所有 TIF 图像。

### 力学 Excel 数据
* 必须包含一列明确的时间列（如 `2026/04/14 21:05:04`）。
* 必须包含您需要提取的变量列（如 `Force`, `Displacement`）。
* 必须为无隐藏sheet状态，格式为.xlsx，如果不是请先导出为Excel文件（.xlsx）。

## 4. 软件操作指南

### 第一步：选择扫描模式
在软件顶部的 **“扫描模式选择”** 区域，根据您的 TIF 数据存放结构，选择 **Type 1** 或 **Type 2**。

### 第二步：设置文件路径
1. **TIF 主文件夹**：点击“浏览”，选择存放 TIF 图像或存放子文件夹的根目录。
   * *注意：选择完成后，软件会自动分析文件夹，并在下方填入扫描参数。*
2. **力学 Excel 文件**：点击“浏览”，选择力学机生成的 `.xlsx` 原始数据表。
3. **结果保存路径**：点击“浏览”，选择一个保存位置，并在弹窗下方输入您期望的**新文件名**（例如 `Result_01`），点击保存。**请勿覆盖原始数据文件！**

### 第三步：核对扫描与时间参数
1. **总扫描次数(n) & 单次扫描TIF数(x)**：由软件自动计算，呈灰色只读状态。请核对日志中打印的数字是否与您的实验预期一致。
2. **时间差 (分钟)**（**极其重要**）：
   * 计算公式：`CT机时间 - 力学机时间`
   * *举例：如果 CT 机显示 18:10，而力学机显示 18:00（CT机比力学机快 10 分钟），则此处填写 `10`。*
   * *举例：如果 CT 机比力学机慢 5 分钟，则填写 `-5`。*

### 第四步：设置 Excel 表头名称
请严格按照力学 Excel 中的表头名称填写（**区分大小写，包含空格或括号也必须完全一致**）：
* **时间列名**：默认为 `Time`。
* **A列名 / B列名**：默认为 `Force` 和 `Displacement`。如果您需要提取扭矩和角度，请修改为 `Torque` 和 `Angle`（需与 Excel 表头绝对一致）。

### 第五步：开始处理
点击 **“开 始 处 理”** 按钮。
软件将在下方日志框中实时打印处理进度。处理完成后，会弹出“数据处理完成”的提示框。

## 5. 结果报告说明

生成的 Excel 结果文件将包含以下列，方便您进行后期核对与分析：

| 列名 | 说明 |
| :--- | :--- |
| **Scan Rank** | 扫描轮次（第几次扫描）。 |
| **Folder Name** | *(仅Type 2)* 数据来源的子文件夹名称。 |
| **CT Start / End Time** | CT 图像的实际生成时间区间（PC2 时间）。 |
| **Mech Target Start / End** | 经过时间差换算后，在力学 Excel 中寻找的目标时间区间（PC1 时间）。 |
| **Data Points Averaged** | 该时间段内，实际提取并参与求平均值的力学数据点个数。 |
| **Average [变量A]** | 该时间段内变量 A（如 Force）的平均值。 |
| **Average [变量B]** | 该时间段内变量 B（如 Displacement）的平均值。 |

## 6. 常见问题 (FAQ)

**Q1：点击“开始处理”后报错：“Excel中找不到表头: 'xxx'”**
* **原因**：您在界面上输入的表头名称与实际 Excel 里的不一致。
* **解决**：打开原始 Excel 文件，检查表头是否有隐藏的空格、大小写是否正确、是否带有单位（如 `Force (N)`）。必须一字不差地填入软件中。

**Q2：日志提示“警告: Scan X 时间段内无数据点，已使用最近单点代替”**
* **原因**：在计算出的 CT 扫描时间段内，力学 Excel 恰好没有记录任何数据（可能是力学机卡顿或采样率极低）。
* **解决**：软件已自动寻找了距离该时间段中心最近的一个数据点作为替代，防止程序崩溃。您可以查看结果 Excel 中的 `Data Points Averaged` 列，如果显示为 `1`，即代表触发了此保护机制。

**Q3：为什么实际 TIF 图片数和软件计算出的理论总数对不上？**
* **原因**：可能存在极少量图片在拷贝过程中丢失，或者多出了几张废片。
* **解决**：软件在日志中会打印黄色警告，但仍会尽最大努力继续处理数据。建议检查原始文件夹，确认数据完整性。
